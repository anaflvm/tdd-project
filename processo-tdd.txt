Uma nova funcionalidade é identificada e o teste funcional para ela é escrito, se o teste funcional não passa ou passa mas em seguida é identificado que a aplicação precisa ser refatorada seguimos para a etapa de escrita do teste de unidade, caso contrário voltamos à identificação de novas funcionalidades.
Na etapa de escrita de testes de unidade um teste é escrito e executado, se o teste de unidade falhar alternamos entre escrever o código mínimo necessário para que o teste passe e a execução do teste, quando o teste passa verificamos se o código precisa ser refatorado. 
A etapa de refatoração seguirá para a escrita de código mínimo necessário para melhorar a qualidade do código mas sem alteração do comportamento, para garantir que de fato não ocorra alteração no comportamento os testes deverão ser executados com sucesso novamente.
